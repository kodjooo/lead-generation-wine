## Актуализация 2026-04-02 — outreach email prompts

- `app/modules/generate_email_gpt.py` теперь формирует короткие письма не про AI-автоматизацию, а от лица крупной розничной сети по продаже алкогольной продукции.
- Системные и пользовательские prompts для email-generation разделены по типам получателей: [email_generation_mall_system_prompt.txt](/C:/Users/Босс001/Для%20переноса/lead-generation-wine/app/prompts/email_generation_mall_system_prompt.txt), [email_generation_mall_user_prompt.txt](/C:/Users/Босс001/Для%20переноса/lead-generation-wine/app/prompts/email_generation_mall_user_prompt.txt), [email_generation_agency_system_prompt.txt](/C:/Users/Босс001/Для%20переноса/lead-generation-wine/app/prompts/email_generation_agency_system_prompt.txt) и [email_generation_agency_user_prompt.txt](/C:/Users/Босс001/Для%20переноса/lead-generation-wine/app/prompts/email_generation_agency_user_prompt.txt).
- `EmailGenerator` больше не подмешивает контекст конкретного сайта в user-prompt для outreach-писем; он только выбирает нужную пару prompts по `entity_type` (`mall` или `real_estate_agency`) и подставляет язык генерации.
- Для `mall` письмо сообщает, что компания рассматривает город для открытия магазина и хочет рассмотреть размещение в конкретном ТЦ, запрашивая условия аренды, доступные площади и коммерческое предложение.
- Для `real_estate_agency` письмо сообщает, что компания ищет помещение под магазин в городе и просит подборку релевантных объектов или коммерческое предложение.
- Prompt и fallback-шаблоны специально ограничены по длине и стилю: деловой нейтральный тон, без ссылок, без кликбейта, без агрессивных sales-конструкций и других признаков массового спама.
- `app/orchestrator.py` подает в генератор обновленный бизнес-контекст `OfferBrief`, соответствующий сценарию аренды помещений, а не сценарию автоматизации процессов.
- Генерация писем поддерживает два транспорта LLM: `EMAIL_GENERATION_LLM_PROVIDER=openai` для прямого вызова `Responses API` и `EMAIL_GENERATION_LLM_PROVIDER=gateway` для вызова внешнего relay-сервиса `/v1/openai/responses` на втором сервере. В gateway-режиме основной проект по-прежнему сам собирает prompt и `responses` payload, а внешний сервис только проксирует запрос в OpenAI.
- Для текущего сценария генерации писем по умолчанию используется `EMAIL_GENERATION_LLM_MODEL=gpt-5` и `EMAIL_GENERATION_LLM_REASONING_EFFORT=low`, чтобы сохранить качество текста, но не разгонять стоимость и латентность на коротких outreach-письмах.
- В очередь на генерацию outreach-писем теперь попадают все primary email-контакты компаний, которые не помечены как `excluded_by_llm`, не имеют `opt_out=TRUE` и еще не имеют `sent` или `scheduled` письма; наличие `homepage_excerpt` и запись в `opt_out_registry` больше не блокируют постановку письма в очередь.
- `app/modules/send_email.py` теперь использует только Яндекс SMTP-конфигурацию `YANDEX_*`: Gmail и MX-маршрутизация удалены из активного контура, а в `outreach_messages.metadata.route.provider` для отправленных писем записывается `yandex`.
- В prompts для outreach-писем добавлен обязательный блок доверия про бренды компании: в каждом письме модель должна кратко и не рекламно упоминать 1 или 2 бренда из набора Винотеки «Культура Крепкого», магазины винзавода «Millstream», каждый раз в разном порядке и без вынесения их в отдельный список, чтобы не ухудшать антиспам-профиль письма.
- Для снижения шаблонности outreach-prompts дополнительно запрещают тяжеловесные обороты вроде `в портфеле форматов` и постоянный заход `Пишу от лица...`, а также заставляют модель чередовать несколько паттернов письма: сначала кто мы, сначала задача по помещению или сначала запрос к ТЦ/агентству.
