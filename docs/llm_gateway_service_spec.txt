ТЗ: внешний LLM gateway как thin relay

1. Цель
Вынести сетевой доступ к OpenAI на второй сервер, не дублируя prompt и бизнес-логику site classification. Основной pipeline должен сам собирать полный OpenAI payload и сам разбирать ответ модели.

2. Что должен делать gateway
- Принимать `POST /v1/openai/chat-completions`.
- Проверять Bearer-токен.
- Принимать OpenAI-совместимый payload с `messages`, `model`, `response_format` и другими полями.
- Отправлять payload в OpenAI без изменения prompt.
- Возвращать raw JSON-ответ `chat.completion`.

3. Чего gateway делать не должен
- Не добавлять свой system prompt.
- Не собирать `site_verdict` из собственного шаблона.
- Не хранить бизнес-правила mall / real_estate_agency.
- Не нормализовать ответ в доменный JSON.

4. Авторизация
- На основном сервере: `SITE_CLASSIFICATION_LLM_GATEWAY_API_KEY`
- На gateway: `GATEWAY_AUTH_TOKEN`
- Формат: `Authorization: Bearer <token>`

5. Endpoint
`POST /v1/openai/chat-completions`

6. Пример payload
```json
{
  "model": "gpt-5-mini",
  "response_format": {
    "type": "json_schema",
    "json_schema": {
      "name": "SiteClassification",
      "schema": {
        "type": "object",
        "properties": {
          "site_verdict": {"type": ["string", "null"]},
          "detected_city": {"type": ["string", "null"]},
          "confidence": {"type": "number"},
          "reason": {"type": ["string", "null"]}
        },
        "required": ["site_verdict", "detected_city", "confidence", "reason"]
      }
    }
  },
  "messages": [
    {"role": "system", "content": "Определи тип сайта по контексту и верни только JSON."},
    {"role": "user", "content": "{\"domain\":\"verno.pro\",\"expected_city\":\"Краснодар\"}"}
  ]
}
```

7. Ответ
Gateway возвращает raw OpenAI JSON:
```json
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "model": "gpt-5-mini",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "{\"site_verdict\":\"official_real_estate_agency_site\",\"detected_city\":\"Краснодар\",\"confidence\":0.94,\"reason\":\"...\"}"
      },
      "finish_reason": "stop"
    }
  ]
}
```

8. Ошибки
- `400` invalid_request
- `401` unauthorized
- `429` rate_limit_exceeded
- `502` upstream_invalid_response
- `504` upstream_timeout

9. Интеграция в текущий pipeline
- `SITE_CLASSIFICATION_LLM_ENABLED=true`
- `SITE_CLASSIFICATION_LLM_PROVIDER=gateway`
- `SITE_CLASSIFICATION_LLM_GATEWAY_URL=https://<gateway-host>`
- `SITE_CLASSIFICATION_LLM_GATEWAY_API_KEY=<token>`

При таком режиме `lead-generation-wine` формирует тот же payload, который раньше отправлял бы в OpenAI напрямую, а gateway выступает только транспортным слоем.
