ТЗ: внешний LLM gateway как thin relay

1. Цель
Вынести сетевой доступ к OpenAI на второй сервер, не дублируя prompt и бизнес-логику site classification. Основной pipeline сам собирает полный OpenAI payload и сам разбирает ответ модели.

2. Что должен делать gateway
- Принимать `POST /v1/openai/responses`.
- Проверять Bearer-токен.
- Принимать OpenAI-совместимый payload с `input`, `model`, `text.format` и другими полями.
- Отправлять payload в OpenAI без изменения prompt.
- Возвращать raw JSON-ответ `response`.

3. Чего gateway делать не должен
- Не добавлять свой system prompt.
- Не хранить бизнес-правила `mall` / `real_estate_agency`.
- Не нормализовать ответ в доменный JSON.
- Не менять `input`, `text.format` или другие OpenAI-поля.

4. Авторизация
- На основном сервере: `SITE_CLASSIFICATION_LLM_GATEWAY_API_KEY`
- На gateway: `GATEWAY_AUTH_TOKEN`
- Формат: `Authorization: Bearer <token>`

5. Endpoint
`POST /v1/openai/responses`

6. Пример payload
```json
{
  "model": "gpt-5-mini",
  "text": {
    "format": {
      "type": "json_schema",
      "name": "SiteClassification",
      "schema": {
        "type": "object",
        "properties": {
          "site_verdict": {"type": ["string", "null"]},
          "detected_city": {"type": ["string", "null"]},
          "confidence": {"type": "number"},
          "reason": {"type": ["string", "null"]}
        },
        "required": ["site_verdict", "detected_city", "confidence", "reason"]
      }
    }
  },
  "input": [
    {
      "role": "system",
      "content": [
        {"type": "input_text", "text": "Определи тип сайта по контексту и верни только JSON."}
      ]
    },
    {
      "role": "user",
      "content": [
        {"type": "input_text", "text": "{\"domain\":\"verno.pro\",\"expected_city\":\"Краснодар\"}"}
      ]
    }
  ]
}
```

7. Ответ
Gateway возвращает raw OpenAI JSON:
```json
{
  "id": "resp-123",
  "object": "response",
  "model": "gpt-5-mini",
  "output_text": "{\"site_verdict\":\"official_real_estate_agency_site\",\"detected_city\":\"Краснодар\",\"confidence\":0.94,\"reason\":\"...\"}"
}
```

8. Ошибки
- `400` invalid_request
- `401` unauthorized
- `429` rate_limit_exceeded
- `502` upstream_invalid_response
- `504` upstream_timeout
